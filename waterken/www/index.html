Configure an SSL key

SSL key pairs for HTTPS must be stored in the keys.jks file. Generate your own key using the following command from the root directory of your installation:

java -jar genkey.jar

If you are behind an HTTP proxy, its address must be specified in the above command. For example:

java -DproxyHost=proxy.example.com -DproxyPort=8088 -DproxySet=true -jar genkey.jar

Access to the network is needed to register the newly generated hostname. The server must be restarted to begin using the new key pair.

Static files

This file, and other static web content, is served from the www/ sub-directory of your installation.

Database creation

Persistent, web-enabled, Java objects are stored under the db/ directory. A new database can be created with a command like:

java -jar share.jar test org.waterken.bang.Main example

The first argument is the name for the new database. The second argument is the fully qualified classname for the first object to construct and export from the database. The third argument is the name of the sub directory containing the corresponding class files.
